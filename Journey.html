04
Chapter Four

v3 — the PDS moment.

The biggest leap: PDS reviews. Specs aren't paperwork — they're conversations. We made the OS host the conversation, with sectioned reviews, inline send-backs, and a paper trail.

Section-by-section review

Reviewers annotate, not approve-or-reject.

Every PDS is broken into sections. Reviewers can flag, suggest, or send back any single section without nuking the whole document.

"I love the formulation. The hero claim is what I want changed."

PDS · sectioned review v3

Version history · revision-aware

Every v remembers why.

Each version is stamped with the inline note that triggered it. The chain reads top-to-bottom like a story: v1 → notes → v2 (addressing notes) → ...
